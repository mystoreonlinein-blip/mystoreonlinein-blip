Onboarding

The onboarding starts after the store has paid for the subscription and has provided all the information that is needed to start processing online orders. We will need details like:

  • GST Number
  • Address of the store
  • Phone number (for the users to allow contacting over a call in case of trouble)

Store QR Code

The store is given a unique QR code that customers can scan to visit your store directly.

Log in

The user can log in using either Google or Apple account. The app itself does not store any password for store owners.

If a given user is associated with more then one stores, the step after log in will prompt to select the store.

Because this is a cloud hosted solution, the users are free to log in from any device they chose.

Delivery enabled?

The store can chose to turn this option on or off.

If this option is turned off, the users can not have the goods dedlivered.

In store pickup enabled?

The store can chose to turn this option on or off.

If this option is turned off, the users can not have the in store pickup.

Store Status & Business Hours

Store Online/Offline Toggle

The store can use the "Store Online" toggle to immediately take the store offline regardless of business hours configuration. When disabled, customers cannot place new orders. The store owner can configure a custom offline message that will be displayed to customers when they try to open the store.

Business Hours Configuration

Configure operating hours for each day of the week independently. For each day, you can choose from three options:

  • Closed: The store does not accept orders on this day. Customers will see "Not accepting orders" for that day.
  • Set Hours: Configure specific opening and closing times (e.g., 09:00 AM to 09:00 PM). Orders are only accepted during these hours, and the store automatically goes offline outside these times.
  • 24 Hours: The store accepts orders all day on this day. Status shows "Open 24 hours".

The system automatically manages store availability based on the configured business hours, turning order acceptance on and off at the specified times each day.

Maximum Pending Orders

Control how many pending orders your store can handle at once to prevent being overwhelmed with orders you cannot fulfill.

How it works:

  • Set to 0: Unlimited orders allowed. The store will accept orders without any cap.
  • Set to a number (e.g., 100): When the number of pending orders reaches this limit, the store will automatically pause and stop accepting new orders. Customers will see a message indicating the store is temporarily overloaded.

Once pending orders are completed and the count drops below the limit, the store automatically resumes accepting orders.

Configure map regions that allow delivery

This configuration controls which areas of the city are servable by the store. Any areas outside that boundary are un-servable for the store.

Our sales team will agree with you on the region and will pass it on to the implementation team.

You will have to reach out to support for changing this option.

Are multiple orders are delivered in a single trip?

If online deliveries are enabled, the store could decide if one dispatch will happen for each order or there will be one larger trip created that will cover all the customers.

Categories

Organize your product catalog into categories to make it easier for customers to browse and discover items.

Features:

  • Create and manage categories: Add categories with names and optional descriptions
  • Nested categories: Support for parent-child category relationships to create subcategories
  • Product tracking: See how many products are assigned to each category
  • Status control: Set categories as Active or Inactive to control visibility to customers
  • Easy management: Edit category details or delete categories as needed

Categories help customers navigate your store efficiently and find products faster.

Product Catalogue

Manage your complete product inventory with detailed product information and flexible pricing options.

Product Information:

  • Basic details: Product name, description, and SKU for inventory tracking
  • Category assignment: Organize products into categories
  • Product images: Add and update product photos
  • Active/Inactive status: Control product visibility to customers

Units and Quantities:

  • Flexible selling units: Support for weight (grams, kg), volume (ml, liters), and custom units
  • Minimum order quantity: Set minimum purchase requirements with automatic unit conversion (e.g., enter as kg or grams, stored as whole grams)

Tiered Pricing (Bulk Discounts):

  • Create multiple pricing tiers based on quantity ranges
  • Define minimum and maximum quantities for each tier
  • Set different prices per unit for each tier to encourage bulk purchases
  • Example: 0.25-1 kg at ₹150/kg, 1+ kg at ₹140/kg
