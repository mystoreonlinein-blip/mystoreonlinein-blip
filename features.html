Onboarding

The onboarding starts after the store has paid for the subscription and has provided all the information that is needed to start processing online orders. We will need details like:

  • GST Number
  • Address of the store
  • Phone number (for the users to allow contacting over a call in case of trouble)

The store is given the unique QR code that would open your store when scanned.

Log in

The user can log in using either Google or Apple account. The app itself does not store any password for store owners.

If a given user is associated with more then one stores, the step after log in will prompt to select the store.

Because this is a cloud hosted solution, the users are free to log in from any device they chose.

Delivery enabled?

The store can chose to turn this option on or off.

If this option is turned off, the users can not have the goods dedlivered.

In store pickup enabled?

The store can chose to turn this option on or off.

If this option is turned off, the users can not have the in store pickup.

Is store offline?

The store can chose to turn this option on or off.

If this option is on, the users will not be able to place any orders at all. This option allows the store to configure an additional message to be shown to the user when the store is offline.

Max pending order

This feature is here to prevent the store from being over loaded from many oders that it can not fulfill

This configuration when set to 0 will allow unlimited number of users to place an order.

This configuration when set to a number will prevent any new orders from coming in. The users will see a message that says that the store is overloaded.

Configure map regions that allow delivery

This configuration controls which areas of the city are servable by the store. Any areas outside that boundary are un-servable for the store.

Our sales team will agree with you on the region and will pass it on to the implementation team.

You will have to reach out to support for changing this option.

Are multiple orders are delivered in a single trip?

If online deliveries are enabled, the store could decide if one dispatch will happen for each order or there will be one larger trip created that will cover all the customers.

Categories

Organize your catalog into categories for easy browsing and faster discovery. Control visibility and ordering.

Product Catalogue

Add, update and disable the products that the user sees.

Add/update the picture.

Support for sizes, colors.

Custom units like mg, gm, KG, ML, Liters, units supported

Configuration of flexible pricing proportional to quantity (e.g. buying 100 KG of something will give you slightly cheaper price).

Live Queue

See orders as they arrive. Accept, prepare, and deliver with real-time status.

The orders can be printed using a printer and handed over for processing.

Invoices & Taxes

Generate itemized invoices automatically with taxes applied correctly.

Customer Profiles

Keep order history and preferences to serve customers better.

Addresses

Save multiple addresses per customer for quick checkout.

WhatsApp Updates

Send confirmations and live updates via WhatsApp to keep customers informed.

Payments

Option to make everything 'pay at the counter'

Option to share the store's UPI id with the users, asking them to provide payment screenshot and later confirming the payment and then confirming the order.

Sales Overview

Track revenue, orders, and trends over time.

Top Products

Identify best-sellers and optimize your catalog.

Enroll delivery agents to the system

Maintain details of the delivery agents in the system.

Enroll them using the email address (for them to log in using SSO).

Assign delivery agent to order(s)

Option to auto assign the order to a single delivery agent.

Lets the owner assign orders to a delivery agent.

If one trip is supposed to deliver many orders, system also optimizes the route. Also, lets the customers know what route an agent is taking without revealing other customers' addresses.
